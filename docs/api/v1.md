# Request API v1

All new backend endpoints are exposed under `/v1`. Responses use the normalized API envelope unless a route explicitly documents a legacy compatibility exception.

## `GET /v1/petition-types`

Lists petition type reference data from the legacy `"petitionType"` table.

### Legacy Mapping

| Legacy route | New route | Source |
| --- | --- | --- |
| `GET /api/v1/petition-types` | `GET /v1/petition-types` | `../request/backend/src/routes/petitionTypes.js` |

### Auth

No authentication is required. The legacy route did not apply auth middleware.

### Rate Limit

The route is limited to 60 requests per minute per client IP, matching the legacy `express-rate-limit` configuration.

### Request

No path params, query params, or request body.

### Success Response

Status: `200 OK`

```json
{
  "ok": true,
  "data": [
    {
      "petitionTypeID": 10,
      "petitionTypeName": "คำร้องทั่วไป"
    }
  ]
}
```

Empty state:

```json
{
  "ok": true,
  "data": []
}
```

### Error Responses

Rate limit exceeded:

Status: `429 Too Many Requests`

```json
{
  "ok": false,
  "error": {
    "code": "RATE_LIMITED",
    "message": "Too many requests."
  }
}
```

Internal error:

Status: `500 Internal Server Error`

```json
{
  "ok": false,
  "error": {
    "code": "INTERNAL",
    "message": "Internal Server Error"
  }
}
```

### Database Behavior

The implementation performs this legacy-equivalent read-only query behavior:

- table: `"petitionType"`
- selected columns: `"petitionTypeID"`, `"petitionTypeName"`
- filter: `"petitionTypeName" IS NOT NULL`
- ordering: `"petitionTypeName" ASC`

The route does not create, update, delete, seed, migrate, or mutate schema.
